Privacy Policy & Data Usage

1. Data Minimization (What we need)

SMFlow is designed to operate on Anonymized, Non-PII (Personally Identifiable Information) data only. To generate a risk score, the software requires only 4 fields:

Required Field Description
Member IDInternal alphanumeric ID (e.g., 10045). No Names.
DateTransaction posting date.
DescriptionMerchant string (e.g., "WALMART POS").
AmountTransaction value.

2. What We Do NOT Collect

We strictly advise clients NOT to input:

  • Real Customer Names
  • Social Security Numbers (SSN)
  • Physical Addresses
  • Date of Birth

3. Local Processing Guarantee

The SMFlow Desktop Application is an Offline Utility. It processes data strictly within the Random Access Memory (RAM) of the local machine. No transaction data is ever transmitted to SMFlow servers, third-party clouds, or external databases. Once the application is closed, the session data is cleared from memory.

Compliance Note: This policy is intended to help Credit Unions maintain compliance with NCUA and GLBA data privacy standards while using third-party analytic tools.
